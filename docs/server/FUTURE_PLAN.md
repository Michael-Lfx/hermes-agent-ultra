# hermes-server 后续计划

## 1. 概述

本文档说明 `hermes-server` 与 Python 版本的功能差异、延后的功能列表，以及延后功能对用户的影响。

## 2. Rust 与 Python 版本差异

### 2.1 已实现功能对比

| 功能 | Python 版本 | Rust 版本 | 差异 |
|------|-------------|-----------|------|
| REST API 框架 | FastAPI | axum 0.8 | 无功能差异 |
| WebSocket | Starlette | axum ws | 无功能差异 |
| JSON-RPC 2.0 | 自定义实现 | 自定义实现 | 协议完全一致 |
| 会话持久化 | SQLite (SessionDB) | SQLite (SessionPersistence) | API 完全一致 |
| 配置管理 | config.yaml | config.yaml | 格式完全一致 |
| Agent 执行 | AIAgent | AgentLoop | 流式回调一致 |

### 2.2 待实现功能差异

| 功能 | Python 版本 | Rust 版本 | 差异说明 |
|------|-------------|-----------|----------|
| Schema 生成 | Pydantic schema | 硬编码 | Rust 暂无 `schemars` 支持 |
| OAuth Web API | 完整实现 | 延后 | 需浏览器交互的 OAuth 流程 |
| Webhook 订阅 | 完整实现 | 延后 | 订阅管理功能 |
| Pairing 管理 | 完整实现 | 延后 | 设备配对管理 |
| Dashboard 主题 | 完整实现 | 延后 | 主题/插件管理 |
| MCP catalog | 完整实现 | 延后 | MCP 目录浏览 |
| PTY 终端 | 完整实现 | 延后 | 浏览器内嵌终端 |
| 事件广播 | 完整实现 | 延后 | /api/pub + /api/events |

## 3. 延后功能列表

### 3.1 PTY 终端桥接

**功能说明**：在浏览器中嵌入完整的 TUI 终端，通过 WebSocket 桥接 PTY 进程。

**影响的用户功能**：
- 浏览器内的终端界面
- 通过浏览器执行 CLI 命令
- PTY 子进程的实时输出

**延后原因**：复杂度高，需要跨平台 PTY 实现（POSIX/Windows）。

**预计实现时间**：Phase 9+（3-4 周）

### 3.2 事件广播（/api/pub + /api/events）

**功能说明**：PTY 子进程通过 /api/pub 发布事件，浏览器通过 /api/events 订阅事件，实现侧边栏的工具调用 feed。

**影响的用户功能**：
- 侧边栏的工具调用实时 feed
- 非聊天界面的事件推送

**延后原因**：依赖 PTY 功能。

**预计实现时间**：Phase 10+（1-2 周）

### 3.3 OAuth / Provider 认证

**功能说明**：Web 界面的 OAuth 登录流程管理（Anthropic PKCE、设备码等）。

**影响的用户功能**：
- 通过 Web 界面添加/管理 API provider
- OAuth 登录流程
- API key 验证

**延后原因**：已有 CLI 实现，Web API 可延后。用户可通过 CLI 或环境变量配置 API key。

**预计实现时间**：Phase 9+（2-3 周）

### 3.4 Webhook 管理

**功能说明**：Webhook 订阅的创建、删除、启用/禁用。

**影响的用户功能**：
- 通过 Web 界面管理 webhook 触发器
- Webhook 事件订阅

**延后原因**：优先级低，用户可通过 CLI 管理。

**预计实现时间**：Phase 10+（1-2 周）

### 3.5 Pairing 管理

**功能说明**：设备配对码的管理（批准、撤销、清除）。

**影响的用户功能**：
- 通过 Web 界面管理设备配对
- DM 配对审批

**延后原因**：优先级低，用户可通过 CLI 管理。

**预计实现时间**：Phase 10+（1 周）

### 3.6 Dashboard 主题/插件

**功能说明**：Dashboard 主题切换、插件管理。

**影响的用户功能**：
- 主题切换
- 插件安装/卸载/启用/禁用

**延后原因**：优先级低，不影响核心功能。

**预计实现时间**：Phase 10+（2 周）

### 3.7 MCP catalog

**功能说明**：MCP 服务器目录浏览和一键安装。

**影响的用户功能**：
- 浏览 MCP 服务器目录
- 一键安装 MCP 服务器

**延后原因**：需要远程目录源，可延后。用户可通过 CLI 手动添加 MCP 服务器。

**预计实现时间**：Phase 10+（1-2 周）

### 3.8 配置 Schema 生成

**功能说明**：生成配置字段的 JSON Schema，供前端渲染表单。

**影响的用户功能**：
- 配置页面的自动表单生成

**延后原因**：需要 `schemars` derive，可延后。初始返回硬编码字段列表。

**预计实现时间**：Phase 9+（1 周）

## 4. 用户影响总结

### 4.1 核心功能（无影响）

以下核心功能不受延后影响：
- ✅ 聊天界面（WebSocket + JSON-RPC）
- ✅ 会话管理（创建、恢复、关闭）
- ✅ 流式 token 输出
- ✅ 工具调用和审批
- ✅ 配置管理
- ✅ 模型切换
- ✅ 技能/插件管理
- ✅ Cron 定时任务
- ✅ Profile 管理
- ✅ MCP 服务器管理

### 4.2 受影响功能

| 功能 | 影响程度 | 替代方案 |
|------|----------|----------|
| 浏览器内嵌终端 | 高 | 使用外部终端 |
| 侧边栏工具 feed | 中 | 聊天界面内显示 |
| Web OAuth 登录 | 中 | 使用 CLI 或环境变量 |
| Webhook 管理 | 低 | 使用 CLI 管理 |
| 设备配对管理 | 低 | 使用 CLI 管理 |
| 主题切换 | 低 | 使用默认主题 |
| MCP 目录浏览 | 低 | 使用 CLI 手动添加 |
| 配置表单自动生成 | 低 | 手动编辑 YAML |

## 6. 总结

`hermes-server` 的 MVP 版本（P0+P1+P2）已覆盖核心功能，可满足大部分用户需求。延后的功能主要涉及浏览器内嵌终端、Web OAuth 登录等高级特性，用户可通过 CLI 或环境变量作为替代方案。

后续版本将逐步补充延后功能，优先级排序为：
1. PTY 终端桥接（用户需求最高）
2. 事件广播（提升用户体验）
3. OAuth Web API（简化认证流程）
4. 其余功能（按需实现）
